Development of Subway Ticketing and Passenger Guidance System

Individual Project,09/2023-12/2023

Project Overview

Due to the production time and purpose, this project currently only supports Chinese. If you want to view it specifically, you can check the code and I will synchronize it to my GitHub. If you have any questions or suggestions, please feel free to let me know

I used a web crawler on the official website of Beijing Subway to obtain the station names of each station, the lines they belong to, and the distance between any adjacent stations. Then, I collected the scenic spots near each subway station in Beijing and mapped them one by one to the subway stations. I also included the scenic spots in the consideration of the initial and final stations, which increased the practicality of the project. I originally used Python to write the backend, mainly for calculating the shortest path between two points using the Dijkstra algorithm. Then, I searched for the average conversion time and average speed of the Beijing subway, which can respectively calculate the shortest time path between any two points and the minimum conversion path. Because the computational complexity is not complicated, this part was written into the frontend using JavaScript.

A page as shown in the figure has been designed in the front-end, which can select the desired destination and current location through a drop-down menu (supports querying), and then users can choose the desired route selection method. After clicking search, you can see the calculated best route in the image and text.
